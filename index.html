Potentially Asked Questions (P.A.Q.)

Is this safe or a ploy to phish account details?
Yes it is safe, no it is not for phishing account details. I put it on GitHub for a reason, feel free to view the horribly effective code yourself and decide.

If authentication details were left around openly on the store I would take my concerns elsewhere.

Is PSDLE meant to compete with or replace the Download List?
It may appear to be a competitor or replacement due to its features. However, I see it as more of a supplement.

While the Download List may have shortcomings, I'm sure those who can officially provide a solution know and do not need to be reminded, told they "do not care", or told how their current product doesn't live up to PSDLE. Thank them for their awesome work instead, they do it for a living!

The Download List is a great thing to have, PSDLE simply makes it better!

Where are all the cool features like PS1/PS2 games, categories, background screenshots, etc.?
Enable "Catalog" during startup to obtain additional information about each game.

Keep in mind this increases the time to generate the list, relative to the amount of items on your list.

Does the search box support regex?
Yes! Simply wrap the regex in /, like usual.
PSDLE supports the /i modifier and a custom /d modifier which excludes matches.
For more information about Javascript Regex, click here.

Does this differentiate between Vita and PS/Vita TV games?
At the time of writing PSDLE can attempt to detect PS TV content for en-us only.
The option will only show up if using the en-us web store, which accepts only en-us accounts.
This is a limitation of the web store, not PSDLE.

If detected, "TV" will appear next to content's platform.

PSDLE:

PSDLE is very powerful and provides a lot of functionality but requires various information, some of which may be personal.

PSDLE accesses the following information when used:
  • Purchase information for generating the list. (PS+ status, time of purchase, etc.)
  • Console activation information. (only if Queue is enabled)
  • Download Queue information. (only if Queue is enabled)
  • Store language. (for determining language-specific things)
Remaining PSDLE functionality is user-selectable (e.g. Catalog) and does not require personal information.

Information collected during PSDLE's runtime is not permanently stored by PSDLE, and never will be. This means anything generated or requested by PSDLE is gone in the e-wind after the resource it was attached to is terminated (e.g. tab). A minor exception to this is the Chrome extension which runs a background task to determine if the store is active or not. If you would like to keep a permanent list please use the "Export View" function.

PSDLE does not transmit any information to an external location. All resources requested or sent (Queue) are in the executed page's context and domain, which is typically the store. Other products or services may interfere with, tamper with, modify, or intercept information PSDLE obtains. This should not be a concern as this information is publicly available in the page context, PSDLE just caches it.
